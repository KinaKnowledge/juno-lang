sys/compile_executableAsyncFunction

Usage

(compile_executableoptions:object)

Description

The compile_executable function generates a stand-alone executable from a source javascript file.  This can be useful for bundling and distributing into a single runtime the application logic and all dependencies.  The output varies on the options object provided.  If no options are provided, the *env_config* object is referenced for default locations and names, and the current permission set is used for the emitted binary.  If successful, an object is returned with sucess set to true and an emitted key indicating where the binary was written.

Options

emit_as
string
A path name to the location to which to write the binary executable file.  By default this value is sourced from *env_config*.exports.save_executable.  If that is not present, then the file is emitted in the current directory with the form `juno.{*env_config*.build}`.  
permissions
array
A list of permissions to be allowed in the emitted executable file.  The current permissions can be accessed with the (permissions) function.  Include only the key names, such as read or net that you wish to retain in the emitted binary.
additional_args
array
A list of command line arguments to be passed to the Deno compile command.
source
string
The source file to be used as the bundled image.  By default this value is taken from *env_config*.exports.save_path 
target
string
The target architecture to emit as.  See `deno compile --help` to see the full list of values available.

sys/dir_treeAsyncFunction

Usage

(dir_treebase_path:stringoptions:?object)

Description

The dir_tree function recursively follows a directory hierarchy and returns a non-nested structure with the contents of the directory and the contained sub-directories.  The returned value is an array containing DirEntry objects with the keys  name, isFile, isDirectory and isSymlink.  
Each DirEntry name value will be the full path to the found entity: the provided base path to the dir_tree function plus the position of the entity in the sub directory structure.
An optional reducer function can be provided to the dir_tree function in options, which will be called with the signature (entry).  If the reducer function returns a true value for the entry when called, the entry will be included in the returned results.  If the reducer returns false, nil or undefined, the entry will be excluded from the final results.  

Example

Find all files that have the ".pdf" extension in a subfolder:
(dir_tree "/tmp"
          {
             `reduce: (function (entry)
                              (and entry.isFile
                                (ends_with? ".pdf" entry.name)))
         })

sys/watch_fsAsyncFunction

Usage

(watch_fspaths:string|array)

Description

Given a path, or an array of paths, creates and returns an FSWatcher.  The provided paths can be files or directories and they must already exist.  When file system events occur in the watched paths (or potentially subpaths), events are then created and returned via the `FSWatcher` object.
Event firing orders are not guaranteed to be returned in chronological order.
Once the path or paths are no longer required to be watched, the FSWatcher object should be closed by calling the close method of the watcher.

Options

recursive
boolean
If true, the subpaths of provided watch directories will also be watched.

Example 

(let
   ((watcher (watch_fs "/tmp")))
   (setTimeout (fn ()
                 (progn
                     (-> watcher `close)
                    (log "closed watcher")))
               10000)
   (for_with (event watcher)
     (log "FS EVENT: " event)))
If a file is then created in the /tmp directory while the block is running, an event will be produced on the console:
FS EVENT:  { kind: "create", paths: [ "/private/tmp/abc.txt" ], flag: null }

sys/with_temp_dirMacro

Usage

(with_temp_dirtmp_path:symbolbody_forms:*)Macro

Description

The with_temp_dir macro creates a temporary named handle which is bound to the tmp_path argument as a first argument to the macro.  The remaining forms (macro arguments) are evaluated in a progn with the symbol name provided to tmp_path bound to the path to the temp directory, which is a string.  As with a progn form, the final form's return value is returned from the evaluated macro code.  Once the provided forms to the macro are completed, the temp directory is removed prior to the return value being returned.  If an exception is encountered during the processing of the provided forms, the exception is caught, the temp directory cleaned up and the exception is then re-thrown.
(with_temp_dir (tt)
         (write_text_file (+ tt "/text.txt") 
                          (+ "TT was: " (is_dir? tt) ": " tt))
        (read_text_file (+ tt "/text.txt")))
